# 採用面接の勉強会MTG
## 第1回
### 決める事
- 目的
  - 面談・面接時にスキルを測る指標(チェックリスト)を作る
  - nendSDKで重要なノウハウを洗い出した上で面接者として皆で考え今後に生かす
    - その手段としてチェックリストを作りたい
  - OS、プラットフォームの知識単位でそれぞれ必要なチェックリストを作るイメージ
- ゴール
  - まずはチェックリストを作る
  - そのステップとして
    - nendSDKで重要なスキル、ノウハウについて全員が理解していること
      - チェックリストの項目の中身を理解しないまま質問だけしていてもあまり見にならない
      - 自分が担当(得意)する領域だけ理解するのがいいのでは？
      - 全員が全領域をカバーするのではなくチームで協力して担当する技術領域も分担するスタンスでよさそう
    - 作ったチェックリストを実際の面接で使うこと
- 保守、運用方法
  - 何を作るか決まってから用意する
    - 候補：github、google driveとか
- ファシリテーター、プロジェクトリーダー
  - 責任者を立ててというのではなく、取りまとめ役としてファリシテーターを決めるので良さそう
  - チェックリストの更新とか管理とかはみんなでやるスタンスで良さそう
  - 上記を踏まえて
    - ファリシテーター：外山さん

  - 次回MTGでブレスト感覚でみなさんから意見を聞く

  - 3課業務に関わるスキルチェックを作る目的のため、まずは現状把握と必要な洗い出しを行い、それらを補足する形で情報収集するスタンス
  - 他社や世間の採用面接でのスキルチェック方法とかを調査するイメージ
  - nendSDK-Document-Privateにissue立てて情報を集めていく

### nendSDK開発(チーム開発)において重要なポイント + 質問

#### 前提知識
- アドテクの知識(業界への知識)
  - 広告SDKを組み込んだ経験はありますか？
    - YES: どこのSDKか？メディエーションの利用経験もあるか？
  - 広告主側に携わった経験がありますか？

- 過去の開発経験(前提の開発力)
  - ブログなどでアウトプットを行っていますか？
  - 勉強会への参加などはありますか？
    - YES: 登壇したことはありますか？
  - ストアへのリリース経験はありますか？
  - OSS活動経験はありますか？

- 複数のOSへの開発意欲(複数OSをまたいだ開発ができるか)
  - サブで他OSの開発した経験はありますか(会社、趣味問わず)
    - 開発だけでなく、レビューも(あるいは片方だけ)経験があるか

- 自ら問題提起をして解決まで先導する経験があるか(リーダー希望、あるいは経験者用)
  - YES:
    - プロジェクトの内容や難易度はどれほどの規模だったでしょうか？
    - 問題への解決はどのように行いましたか？

- ユーザーとのお問い合わせ等、社内外とのやり取り経験があるか(コミュニケーション力、プロダクトへの理解力)
  - お問い合わせなどでユーザーの問題を解決に導いたことがありますか(社内外問わず)
    - 具体的にどういった内容だったか出来れば説明して下さい
    - どういった問い合わせ形式でしたか？(電話、チャット、メール)

- 英語のドキュメントを読むことに耐性がある人(同文)
  - 公式サイト(英語)やGitHubのリポジトリにあるフレームワークのドキュメントをもとに学習してアプリに導入した経験はありますか？

- トレンドに対する学習ができているか(最新の技術に対して敏感でプロダクトへの導入検討まで出来ること)
  - 最近の流行っているアプリを触っているか
    - アプリのどういった点を重点的に見ているなどありますか？
      - UI,UXデザイン？
      - 分野(ブロックチェーン、ゲーム系など)
  - 最新のフレームワークに触れた経験が直近でありますか
    - iOS: SwiftUIなど
    - Android: Jetpack Composeなど
  - 普段どのようにトレンドを追っていたりしますか？
    - 回答例1: 公式サイトのニュースをRSSで購読
    - 回答例2: 界隈の著名人のブログからトレンドに対する所感などを見ている

#### 共通(チーム開発)

- 社内ドキュメントの作成経験(長期的なプロダクトを見据えたフローで開発できていたか)
  - 仕様書などを作成した経験がありますか
    - YES: 何でドキュメント化していましたか？(GitHub wiki?, Google drive)

- チーム開発の経験(チーム開発の基本を知り得ているか)
  - コードレビューした経験があるか
  - 他の開発チームと連携してタスクを進めたことがあるか(仕様決め、相談等)
    - チーム規模はどのくらいのものでしたか

- ソースコードの管理システムの使用歴(Gitを使えるのか)
  - GitHubなどを利用してソースコードをリモートで管理したことがありますか
  - Pull requestを作成したことがありますか？

- テレワーク環境に耐性があるか
  - Slackなどを用いてテキストコミュニケーションを中心に行ったことがあるか(テキストコミュニケーションが中心の文化にすぐ慣れることができるか)

#### 共通(開発)
- CI/CD等のツールの経験
- デバッグに必要な知識、経験等
- ライブラリの開発経験
- スマホアプリに関わらず、サーバーサイドなどの開発経験はあるか
- obj-c,java,swift,kotlinなどnendSDKに関わる言語の使用歴
- クロスプラットフォーム開発の知識
- テストの経験
- パフォーマンスを意識した開発をしたことがあるか
- 新OSの対応経験

#### Android

- Gradleのビルドシステムの知識
- javaスレッドの仕組み、それに関するデザインパターンなど
- タブレットとスマホでそれぞれ意識してアプリ開発したことがあるか

#### iOS

- XCodeのビルドシステムの知識
- スレッドの仕組み、それに関するデザインパターンなど
- CocoaPodsなどパッケージマネージャーの知識
- iPadとiPhoneでそれぞれ意識してアプリ開発したことがあるか
